Defining new effects

Non-pure side-effects in F* are modelled through computation types, also known as effects. We take a look at how to define and use new custom effects to represent state in an external system. The example we use is a set of valves controlling the flow of water into and out of a water tank.

Using OCaml libraries in F*

Since F* has a small ecosystem, there are few libraries written in F*. This means that larger projects will likely need to interact with code written in other languages. Fortunately, OCaml has a large ecosystem which we can use from F*.

To illustrate how this works, we will write an F* program which uses the OCaml Yojson library to serialize and deserialize JSON.

Setting up an F* development environment

There are several ways of setting up a development environment for F*. The approach I prefer is to build F* itself from source, but use the version of opam provided by the system package manager, and to use the F* mode for Emacs to edit F* files.
